Highlights

Launch software from cloud to your local machine in one click. To launch software in Windows, a batch file is provided. But, there are two requirements that should be fulfilled before launching the software using batch file.

  1. Windows does not have X11 window server, by default. So, we are using a 3rd party X11 windows server, VcXsrv that should be installed.

  2. Also, another software, PUTTY is used to SSH to the cloud, should also be installed.

Batch File firefox.bat:

cd C:\Program Files\VcXsrv
xlaunch.exe -run config.xlau
cd C:\Program Files\PuTTY
putty.exe -X -C -ssh root@192.168.43.98 1235 -pw q -m test.txt

Note:

  • Launching VcXsrv from command line launches GUI that asks for configuration steps to be done. So, we saved the config manually to a file config.xlau and start the program using that config file.

  • There is also a requirement of another file, test.txt which contains the list of commands to be ran automatically on SSH. In this case, it just contains Firefox. For now, we have manually put it into PuTTY installed location.

Problems

Problem: While providing Firefox using SaaS, the memory consumption continues to increase. Soon, it consumes all of the available RAM and system hangs.

Solution: While launching Firefox or any another program in a container, we can limit it's memory consumption by launching it using this command: ulimit -Sv 512000 && firefox
